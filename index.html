Coming Full . . . Table?

CSS Grid is treated as an advanced feature. This is despite its having been around for almost three years now, and despite its simplicity and usefulness as a layout engine. In large part, however, CSS Grid's niche appearance seems to be due to its lack of usage in major frameworks: Bootstrap, for example, does not use CSS Grid, preferring instead to construct its own pseudo-grid with SCSS. In other words, CSS Grid today seems to be treated as a bit of advanced vanilla CSS that doesn't really find a place in Bootstrap-built application CSS environments.

This is a pity. In terms of structure and design, not only is CSS Grid a jewel of a layout engine, it also hearkens back to the Internet's early days, and the first layout engine ever employed in web development -- HTML Tables.

A screenshot of the Dole-Kemp website, from qz.com

The first Web browser to find popular appeal was Mosaic in 1992. This would be followed by Netscape Navigator in 1995. During its peak in the late 1990s, Navigator introduced millions worldwide to the Web, but, at the time, the only design tools widely available were those HTML itself offered. It did not take developers long to realize that application-level HTML pages were a maintenance nightmare, with the same standards needing to be hand-coded again and again within the document ... and that HTML seriously lacked tools as a layout engine.

By the late 1990s, as Internet Explorer was gaining market share over Netscape Navigator, web-design pioneers made a key discovery: HTML tables could be hacked to construct the first modern web pages. While we might look at Bob Dole's 1996 campaign website (he lost) as a period relic today, back when it was released in 1996, it was revolutionary. Its creators made use of Microsoft FrontPage, a WYSIWYG layout editor that was only just becoming popular along with early versions of Internet Explorer, and in its source code, we find ... tables. Loads and loads of tables.

The Dole-Kemp page's source code. Note the lack of CSS and JavaScript (both specifications were new at the time), the use of HTML <tr> and <td> elements setting tables, and that the code was written on FrontPage 6.

Soon, the use of tables utterly dominated the world of Internet design. By the early 2000s, the vast majority of the web was laid out with HTML tables. This, however, did not sit well in some circles, who viewed such widespread use of tables for clearly non-tabular purposes as a gross misuse of the tool. However, by the mid-2000s, with screens getting bigger and browsers getting faster, that classic early-2000s HTML tables-driven look had become pass√© and dated. Even so, neither HTML nor CSS offered anything close to a reasonable design engine at the time. Web designers began to experiment with CSS hacks like floats and absolute positioning, and designers with preprocessor experience began to build the progenitors of Bootstrap "grid's" columnular layout engine.

However, there was a baby in the bathwater, and this baby would be expressed in CSS Grid. The reason HTML tables were such an effective design tool in their day is because a table is, fundamentally, a grid. At its heart, CSS Grid works by creating, on the parent element, a table that daughter cells can nest in. It does this more powerfully and agilely than HTML tables ever could, too.

An example of CSS Grid nesting. Note that objects E, F, and G are a subgrid inside object D.

When I built my drum kit, and when I built my random quote generator, I used CSS Grid extensively. And why not? It is a fully two-dimensional layout engine. Not only can I set items in the grid, I can use media queries to reset the grid itself at breakpoints in order to yield a fully responsive web app. I can even set grids inside of other grids in order to construct more advanced design schemata. (Indeed, my drum kit employs such a thing extensively.)

Grids can be hard to pull off for a CSS novice who hasn't yet mastered how to stretch boxes out to the dimensions they need to be. Their learning curve is certainly steeper than Flexbox's. That said, calling CSS Grid "advanced" oversells its difficulty. At the end of the day, CSS Grid, like Flexbox, is simple, powerful, and intuitive. It is for us, today, what HTML tables were like for the web design pioneers of the mid-1990s: a tool that can change how we think about the very webpages and web apps we design, and design them to be simpler, cleaner, neater, and more maintainable.
